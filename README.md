# ☁️ droplet-bootstrap

## Overview

Infrastructure as code for DigitalOcean. Spin up a droplet with Terraform, configure and harden it with Ansible.

This repository contains all the scripts, configurations, and descriptors needed to manage and automate the
provisioning and configuration of DigitalOcean Droplets. Whether you're interested in setting up a similar
environment, learning about infrastructure automation, or just browsing through some useful scripts, you'll find
everything organized and ready to use.

Feel free to explore, contribute, or adapt the resources to fit your own needs!

> ⚠️ Remember this is just for fun, do not apply these configurations to set up a Production environment 😀

---

## 🧅 Stack

- [DigitalOcean](https://www.digitalocean.com/) as the cloud provider.
- [Terraform](https://www.terraform.io/) (>= 1.5.0) to provision Droplets.
- [Ansible](https://www.ansible.com/) (>= 2.14) to configure and harden Droplets.
- [pre-commit](https://pre-commit.com/) for linting and security checks.

## 🛡️ Security

The playbook applies the following hardening measures to the droplet:

| Layer | Measure |
|---|---|
| **SSH** | Password auth disabled, root login disabled, `AllowUsers deploy`, MaxAuthTries 3 |
| **Users** | Non-root `deploy` user with SSH key + passwordless sudo, root locked out of SSH |
| **Firewall (UFW)** | Default deny incoming, allow 22/80/443 only |
| **Firewall (DO)** | Optional `ssh_allowed_ips` to restrict SSH at the cloud level |
| **fail2ban** | SSH jail: 3 attempts in 10min = 1h ban |
| **Kernel** | SYN flood protection, reverse path filtering, ICMP redirect/source route disabled, martian logging |
| **Updates** | Automatic daily security patches via unattended-upgrades |
| **Filesystem** | `/run/shm` mounted noexec/nosuid/nodev, core dumps disabled |
| **Shell history** | Commands containing TOKEN/SECRET/KEY/PASSWORD auto-excluded, space-prefixed commands ignored |
| **IaC** | Providers pinned to exact versions, `.terraform.lock.hcl` committed, detect-secrets hook |
| **SSH banner** | Legal warning displayed before login |

## 🚀 DIY

### Prerequisites

- [Terraform](https://developer.hashicorp.com/terraform/install) >= 1.5.0
- [Ansible](https://docs.ansible.com/ansible/latest/installation_guide/) >= 2.14
- [pre-commit](https://pre-commit.com/#install) >= 3.7.0
- A [DigitalOcean](https://cloud.digitalocean.com/) account with an API token
  (see [required scopes](#-api-token-scopes) below)
- An SSH key registered in your DigitalOcean account

### Setup

**1. Clone the repo**

```bash
git clone https://github.com/your-user/droplet-bootstrap.git
cd droplet-bootstrap
```

**2. Install dependencies**

```bash
pip install -r requirements.txt
ansible-galaxy install -r ansible/requirements.yml
pre-commit install
```

**3. Configure your variables**

```bash
cp terraform/terraform.tfvars.example terraform/terraform.tfvars
# edit terraform.tfvars with your values
```

Available variables:

| Variable | Description | Default |
|---|---|---|
| `region` | DigitalOcean region | `ams3` |
| `droplet_size` | Droplet size slug | `s-1vcpu-1gb` |
| `droplet_image` | OS image slug (ignored when `snapshot_id` is set) | `ubuntu-22-04-x64` |
| `snapshot_id` | DO snapshot ID to create droplet from | `""` (skip) |
| `droplet_name` | Name for the droplet | `my-server` |
| `ssh_key_fingerprint` | Name of SSH key in DO | *required* |
| `ansible_user` | SSH user after hardening | `deploy` |
| `ansible_ssh_private_key` | Path to local SSH key | `~/.ssh/id_rsa` |
| `ssh_allowed_ips` | CIDRs allowed to SSH | `["0.0.0.0/0", "::/0"]` |
| `project_name` | DO project to assign droplet to | `""` (skip) |
| `tags` | Tags for the droplet | `["terraform", "ansible"]` |

**4. Export your DO token**

```bash
export TF_VAR_do_token="dop_v1_xxxxxxxxxx"
```

**5. Deploy 🚀**

```bash
./scripts/deploy.sh
```

The script will:

1. Run `terraform init`, `plan`, and `apply` to create the droplet
2. Wait for SSH to become available
3. Run the Ansible playbook as root to create the `deploy` user and harden the machine
4. Print the SSH command to connect as `deploy`

### Teardown

```bash
./scripts/destroy.sh
```

The script will ask for confirmation, then destroy all resources
(droplet, firewall, project assignment) and clean up the generated inventory file.

## 🔑 API Token Scopes

When creating your DigitalOcean API token, use **Custom Scopes** with the following permissions (20 scopes total):

| Access level | Resources |
|---|---|
| **Fully scoped** | `actions` (read), `firewall` (create, read, update, delete), `regions` (read), `sizes` (read) |
| **Create** | `droplet`, `project`, `tag` |
| **Read** | `droplet`, `image`, `project`, `snapshot`, `ssh_key`, `tag`, `vpc` |
| **Update** | `droplet`, `project` |
| **Delete** | `droplet` |

> 💡 Alternatively, you can use a **Full Access** token if you prefer simplicity over least-privilege.
