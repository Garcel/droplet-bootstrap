# ☁️ do-infra

## Overview

Infrastructure as code for DigitalOcean. Spin up a droplet with Terraform, configure it with Ansible.

This repository contains all the scripts, configurations, and descriptors needed to manage and automate the provisioning and configuration of DigitalOcean Droplets. Whether you're interested in setting up a similar environment, learning about infrastructure automation, or just browsing through some useful scripts, you'll find everything organized and ready to use.

Feel free to explore, contribute, or adapt the resources to fit your own needs!

> ⚠️ Remember this is just for fun, do not apply these configurations to set up a Production environment 😀

---

## 🧅 Stack

- [DigitalOcean](https://www.digitalocean.com/) as the cloud provider.
- [Terraform](https://www.terraform.io/) to provision Droplets.
- [Ansible](https://www.ansible.com/) to configure Droplets.

## 🚀 DIY

### Prerequisites

- [Terraform](https://developer.hashicorp.com/terraform/install) >= 1.5.0
- [Ansible](https://docs.ansible.com/ansible/latest/installation_guide/) >= 2.14
- A [DigitalOcean](https://cloud.digitalocean.com/) account with an API token
- An SSH key registered in your DigitalOcean account

### Setup

**1. Clone the repo**

```bash
git clone https://github.com/your-user/homelab.git
cd homelab
```

**2. Configure your variables**

```bash
cp terraform/terraform.tfvars.example terraform/terraform.tfvars
# edit terraform.tfvars with your values
```

**3. Export your DO token**

```bash
export TF_VAR_do_token="dop_v1_xxxxxxxxxx"
```

**4. Deploy 🚀**

```bash
./scripts/deploy.sh
```

Terraform will provision the Droplet and generate `ansible/inventory/hosts.ini` automatically. Then Ansible will kick in and configure the machine.

### Teardown

```bash
cd terraform && terraform destroy
```
